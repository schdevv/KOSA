### 1. Search Bot

#### 정보검색개요
```
 1) Data 종류
  ● 정형 데이터
  ● 반정형 데이터
  ● 비정형 데이터
   - 정의된 구조가 없이 저장된 데이터
   - 정형데이터를 색인해 비정형데이터로 사용하는 방식
   - 소셜 데이터의 텍스트, 이미지, 영상, 워드, PDF 등
   
 2) 검색시스템
  ● 데이터 검색 : 구조화된 Database에서 특정 record를 검색
   - 예시: 보스턴에 있는 일본음식점 방문객들의 방문 횟수에 대한 추세를 보여줘
  ● 정보 검색 : 비정형 데이터를 Document에서 찾아냄 
   - 예시: 일본 식당에서 가장 많이 발생하는 식중독 유형에 대해 알려줘

 3. RDBMS(관계형 데이터베이스) vs 검색엔진
  ● RDBMS
   - SQL을 이용해 정확한(키워드와 매칭되는) 데이터만 검색할 수 있음.
   - 텍스트를 여러단어로 변형하거나 동의어, 유의어 활용 검색이 불가능.
   - 비정형 데이터의 색인과 검색이 불가능
  ● 검색엔진
   - 텍스트를 여러 단어로 변형하거나 동의어, 유의어 활용 검색 가능
   - RDBMS에서 불가능한 비정형 데이터의 색인과 검색 가능

 4. 정보검색 시스템 원리
  ●   문서집합 → 색인모듈 → 질의 키워드 → 색인어 추출 → 색인어의 출현 빈도 계산 → 색인어를 키워드로하는 색인 생성 → 색인어와 연관된 문서 리스트 생성 → 질의와 색인어를 비교하여 검색 결과 도출
 
 5. 문서표현  
  ● 단어들의 순서는 전혀 고려하지 않고, 단어들의 출현빈도(frequency)에만 집중하는 텍스트 데이터의 수치화 표현 방법, 자연어처리나 정보 검색(IR)에서 사용.
   - document
   - 형태소 분석
   - 품사 태깅
   - 불용어 제거
   - 복합명사 분해
   - 색인어 추출
   - 필요없는 조사 등 데이터를 추림    
  ● Bag of words : 단어들의 순서는 고려하지 않고 단어들의 출현 빈도에만 집중하여 텍스트를 수치화하는 방법
   - 형태소 분석, 품사 태깅, 불용어 제거, 복합명사 분해, 색인어 추출 등의 과정을 거쳐 적용

 6. TF/IDF : 특정 단어를 검색한 키워드를 랭킹
  - TF와 IDF의 곱으로, 특정 단어를 검색한 키워드의 랭킹을 결정하는 데 사용
  ● TF는, 하나의 Document에서 1이 될 수 있다.
  ● DF는, 하나의 Document에서 1만 될 수 있다.
  ● IDF(Inverse Document Frequency) : 특정 단어가 얼마나 문서 집합에서 희귀한 단어인지를 나타내는 지표

 7. 프로세스 : Data 수집& 정제 -> 형태소 분석 -> 감성어 사전 제작 -> 모델링 -> 결과분석
```

#### 정보검색시스템
```
 1) 정보검색 시스템 구성
  ● 인덱싱 과정 : 원시 정보 수집 → 정보 필터링 → 언어처리(형태소분석/stem) → 인덱싱 키워드 추출 → 인덱스 파일 생성
  ● 언어자원 : (인덱싱 과정)언어처리 → (질의/검색과정)[질의분석 → 질의 확장]
  ● 인덱스 파일 : (인덱싱 과정)인덱스 파일 생성 → (질의/검색과정) 인덱스 연산
  ● 질의/검색 과정 : 정보 요청 → 질의 분석 → 질의 확장 → 인덱스 연산 → 순위화(랭킹)
   - 동일어 사전

 2) 인덱싱 과정
  ● Tokenization
   - 문서를 토큰화 하는데 
  ● 토큰화 수행시 고려할 사항

  ● stop word : 색인어로서 가치가 없다고 판단되는 단어(문서)에 포함되는 단어
  ● Normalization : 단어로부터 접사 등을 제거하고 정규형으로 변환하는 과정
   - 빈도수를 정확히 분석함 cooking, cooked, cook, sasy → cook :3 , easy : 1 
  ● Stemming vs Lemmatiztion

 2) Inverted File(DOR)
  ● .tis : 첫 검색시 이정보를 이용하여 필드별 정렬정보 생성(TermDoc)
   - 과정 예시 : 저장 / 전처리 / user

 3) Inverted Index(역색인)
  ● 색인 (Index)
   - 특정한 데이터의 위치를 미리 저장해, 검색 시 빠른 속도로 찾을 수 있도록 함.
   - 데이터셋이 많은 경우, 1~50만번까지 순차대로 검색.
  ● 역색인 (Inverted Index)
   - 데이터 색인 시 인간의 사고와 가깝게 체계를 구성함.
   - 어떠한 데이터를 색인할 때 Term(단어) 기준으로 색인을 수행
   - fox를 포함하고 있는 문서의 id를 바로 알 수 있음.    
  ● 프로세스 
   - Indexing(구문 분석, 구문별 형태소)-> Inverted Indexing -> Posing File

 4) 질의/검색 과정
  ● 질의 분석 및 연산
   - 형태소 분석 : 나는 학교에 갑니다
    - n-gram : 입력한 문장, 키워드를 두 글자로 잘라 형태소를 분석
     - 나는 / 학교 / 에갑 / 니다
     - tri-gram :
     - by-gram : 
     - 재현율, 정확률
     - AND, OR 검색
  ● 질의 확장
   - 유의어 사전
   - 시소러스
  ● 인덱스 파일 검색
   - Boolean
   - Prefix
   - Near
   - Phrase
  ● 검색 결과 정렬
   - 검색 엔진에서 정렬이 중요함.
   - TF/IDF에 따른 재정렬
   - 어휘 통계적 정보, 웹의 링크와 같은 구조적 정보
  ● 검색 결과 표현 

 5) Boolean Model
  ● Query : 피연산자인 용어들을 불리언 연산자로 표현
   - 솔트룩스 AND 톡봇, 톡봇 OR 챗봇
  ● Document : 용어들의 집합으로 표현
  ● Retrieval : 각 Query Term T에 대해 T를 포함하는 문서집합 ST에 대응
   - 
  ● 예시: Query = 북한 AND 미사일 AND 미국
   - 검색결과 :  R = S 북한 AND S미사일 AND S미국 = (D6) 
  ● 권한처리 : 1부터 10까지 색인한 필드에 권한 코드 2,3만 가지고 사옹자에게 제공함
   - 2와 3은 OR로 묶어 AND 연산해 결과를 리턴해줌
   - 여러 조건을이 뒤섞이는데 이를 해결하기 위해 활용함.

 6) SCORE Model : 검색결과 정렬 모델
  ● BM25 (Best Match 25)
   - 주어진 쿼리에 대해 문서와의 연관성을 파악
   - Baog-of-words 개념을 사용해 쿼리에 있는 용어가 각각의 문서에 얼마나 자주 등장하는지를 평가함
   - 수식
   - f(qi,D) : 쿼리의 i번째 토큰이 문서 D에 얼마나 자주 나타나는가? 
   - IDF(q_i) : 쿼리의 i번째 토큰에 대한
   - IDI/avgdi : 문서의 길이에 대한 부분

 7) 검색의 품질 (Recall/Precision)
  ● Recall(재현율) : 모델이 참으로 분류해야 할 전체 샘플 중에서 모델이 참으로 분류한 샘플의 비율
  ● Precision(정확률) : 모델이 참으로 분류한 전체 샘플 중에서 실제로 참인 샘플의 비율
  ● AND 연산 : 두 개 이상의 검색어를 조합한 경우, 모든 검색어가 포함된 문서를 검색
   - 예시 : "apple" AND "orange"를 검색하면 "apple"과 "orange" 단어가 모두 포함된 문서가 검색됨.
  ● OR 연산 : 두 개 이상의 검색어를 조합한 경우, 하나 이상의 검색어가 포함된 모든 문서를 검색
   - 예시 :  "apple" OR "orange"를 검색하면 "apple"이 포함된 문서나 "orange"가 포함된 문서가 검색됨.

```

#### DISCOVERY
```
 1) 검색엔진 DISCOVERY
  ● 실시간 비정형 분석과 검색
   - 클라우드 기반 시멘틱 검색 및 마이닝 엔진

 2) 특징
  ● 시멘틱 통합검색
  ● 강력한 비정형 데이터 분석(텍스트 마이닝)
  ● 패싯 필터와 강력한 질의 처리  
 
 3) 용언 색인(동사/형용상 등)
  ● 비정형 문서를 분석하여 형태소 정보를 각 문서에 태깅하고, 해당 정보를 활용하여 다양한 형태의 감성부석이 가능한 검색 기능을 제공

 4) 불리언, 자연어 검색
  ●  불리언 : AND, OR, NOT과 같은 논리 연산자를 사용하여 검색어를 구성하고, 이를 이용하여 검색하는 방법
  ● 자연어 검색 : 일상적으로 사용되는 언어로 검색어를 입력하는 방법

 5) 고급검색 - 상세검색 / 결과 내 검색 

 6) 최근/인기검색어

 7) 검색 결과 캐시 기능

 8) 유니코드 지원
 
```

 2) 파인튜닝
  ● downstream : 데이터 처리 과정에서의 다음 단계를 의미함. 
   - upstream 처리 후에 downstream 처리가 이뤄짐.

 3) langchain gpt
  ● 프롬프트를 달리해 여러번 되묻고 적절한 결과를 찾아가는 방법
  ● 다음 답변 중 골라줘 그런식
  ● 프롬프트 10개
  ● 경험기반의 방법론

 4) 자연스러운 대화 방법
  ● 프롬프트(Prompt) : 명령어를 입력하기 위한 화면 상의 권유 메시지나 텍스트
  ● 컨플리션(Completion) : 명령어나 파일 이름 등을 자동 완성해주는 기능
